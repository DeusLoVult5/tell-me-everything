# 完整访谈指南

## S 级模板 — 闸门（S1-S3，所有模式必经）

```
S1 — 隐私与安全
  问题："项目是个人的还是公司的？涉及敏感数据吗？能上传代码/调外部API吗？"
  选项（单选）：
    个人项目，无限制
    个人项目，有限制（需说明）
    公司项目，有合规要求

S2 — 项目边界
  问题："AI 能碰哪些文件？不能碰哪些？"
  选项（单选）：
    限定项目目录内
    可读工作区不写
    工作区内自由
    别碰代码之外的文件

S3 — 操作确认阈值
  逐项确认（单选 ×7）：
    读文件 — 自动 / 先问
    写文件 — 自动 / 先问
    删文件 — 自动 / 先问
    跑命令 — 自动 / 先问
    Git操作 — 自动 / 先问
    安装依赖 — 自动 / 先问
    确认有效期 — 仅本次 / 本次会话 / 永久
```

## S 级模板 — 后半（S4-S6，full 模式）

```
S4 — Env 环境路径
  Q: "conda环境名叫什么？需要我查一下吗？"
  Q: "项目放哪个目录？"
  Q: "文件来源在哪？你怎么给我？"
  → S 级不能代定

S5 — 确认后有效期（依附于 S3）
  → S3 的"确认有效期"答了才追问

S6 — 环境操作"谁来"（依附于 S4）
  → S4 答了才追问：新建 conda / 装依赖 / 改配置 分别谁来
```

### A 级模板（按需追问）

```
A 级第一问 — A10 参考优先（S 级过后首个提问，不可跳过）

  问题："有没有参考项目或你见过类似效果的东西？"
  选项（单选）：
    有，我发给你（提供链接/文件/截图）
    有，但不好找
    没有，帮我搜

  有参考 → 优先对齐参考项目效果
  没有 → Agent 搜索（官方文档 > 知名开源项目 > 个人博客）
    → 找到展示确认 / 搜不到透明告知风险
  个人博客方案 → 标注来源未经验证

---
根据 S 级答案决定追问以下哪些：
  练手 → 跳过部署/CI/CD
  上线 → 追加部署/安全/监控
  纯前端 → 不问后端框架
  单人 → 跳过协作/PR

I9  版本控制（单选）
I4  项目上下文（单选）
I5  技术栈（多选）
I6  项目架构（多选）
I7  工具链（多选）
I11 测试策略（多选）
I12 三方库规范（单选）
I13 错误处理（单选）
```

## "我不懂"处理协议 — 卡片

```
用户说"我不懂" →
  ✅ 降低门槛（术语→白话，场景→具体）
  ✅ 给锚点（"一般是 A 或 B"）
  ❌ 不替用户做决定

还答不上来 → "我先用最常见方案，优点X，风险Y。你记着有Y的风险。"
只有明确"你帮我选" + M5确认 → 才能代选
```

## 访谈规则 — 卡片

```
- 一个弹框只问一个决策。what和who分两个问题
- 互斥选单选。可组合选多选
- 多选 ≤4个选项。超则拆子问题
- S级和A级不合并在同一弹框
- 环境/路径/文件来源 → S级，不能代定默认
```
