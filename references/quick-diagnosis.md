# 快捷诊断 + 完整访谈指南

## Quick Mode — 3 问快捷诊断

```
Agent 弹 AskUserQuestion（3个问题，一次一个）：

Q1: 纯前端还是前后端？
    选项：纯前端（无后端） / 前端+后端 / 不确定，帮我分析

Q2: 数据存哪？
    选项：本地（浏览器/文件） / 云端（服务器/数据库） / 不确定

Q3: 这个项目到什么程度？
    选项：练手/课程作业 / 自用 / 上线给别人用 / 不确定

用户答完 →
  自动诊断 → 弹框：

  "快速分析：{基于3个答案的技术路线}
   风险点：{2-3个}
   建议 Phase 1：{技术栈+交付物}

   接下来："
  ├─ "完整初始化" → 切 full
  ├─ "先出方案"   → 基于默认参数直接出路线图
  └─ "就按这个来"  → 写 CLAUDE.md→切 IMPLEMENT
```

如果用户连 3 个问题都答不上来 → "我不懂"处理协议。

## Full Mode — 全维度访谈

### S 级模板（必须全过）

```
I1 — 隐私与安全
  问题："项目是你个人的还是公司的？代码能上传吗？能调外部API？"
  选项（单选）：
    个人项目，无限制
    个人项目，有限制（说明）
    公司项目，有合规要求

I2 — 项目边界
  问题："AI 能碰哪些文件？不能碰哪些？"
  选项（多选）：
    限定项目目录内
    可读工作区，不写
    工作区内自由
    别碰代码之外的文件

I3 — 操作确认阈值（分项单选）
  Q: 读文件 — 自动 or 先问？
  Q: 写文件 — 自动 or 先问？
  Q: 删文件 — 自动 or 先问？
  Q: 跑命令 — 自动 or 先问？
  Q: Git操作 — 自动 or 先问？
  Q: 安装依赖 — 自动 or 先问？
  Q: 确认有效期 — 仅本次 / 本次会话 / 永久

Env — 环境路径
  Q: "conda环境名叫什么？需要我查一下吗？"
  Q: "项目放哪个目录？"
  Q: "照片/参考文件在哪？你要怎么给我？"
  → S 级不能代定
```

### A 级模板（按需追问）

```
根据 S 级答案决定追问哪些：
  练手 → 跳过部署/CI/CD
  上线 → 追加部署/安全/监控
  纯前端 → 不问后端框架
  单人 → 跳过协作/PR

I9  版本控制（单选）
I4  项目上下文（单选）
I5  技术栈（多选）
I6  项目架构（多选）
I7  工具链（多选）
I11 测试策略（多选）
I12 三方库规范（单选）
I13 错误处理（单选）
```

### B 级模板（用户说"更多"才展开）

```
I8  仓库边界（多选）
I10 编码风格（多选）
I14 日志/留痕（分项确认，A级——硬门）
I15 AI交互偏好（单选）
```

## "我不懂"处理协议 — 卡片

```
用户说"我不懂" →
  ✅ 降低门槛（术语→白话，场景→具体）
  ✅ 给锚点（"一般是 A 或 B"）
  ❌ 不替用户做决定

还答不上来 → "我先用最常见方案，优点X，风险Y。你记着有Y的风险。"
只有明确"你帮我选" + M5确认 → 才能代选
```

## 访谈规则 — 卡片

```
- 一个弹框只问一个决策。what和who分两个问题
- 互斥选单选。可组合选多选
- 多选 ≤4个选项。超则拆子问题
- S级和A级不合并在同一弹框
- 环境/路径/文件来源 → S级，不能代定默认
```
