# 门控卡片

## M5 状态门 — 6 步卡片

```
步骤0 — M0追问
  缺要素（哪/做了什么/什么现象/期望什么）→ 不问完不动手

步骤1 — 规模评判
  Agent陈述事实 → 弹 AskUserQuestion → 用户确认级别

步骤2 — 写入 CLAUDE.md
  【当前模式：IMPLEMENT】
  【规模：X改动 — N文件】
  【预计改动：file1 / file2】
  【验证方式：...】

步骤3.5 — 自检
  逐条过 Pre-Implementation 自检清单（8条）。
  全部✓ → 进入步骤4。任一条✗ → 停下处理。

步骤4 — 实现
  在已确认的边界内写代码

步骤5 — M7+日志
  /review 审查 → 写日志（logs/agent/YYYY-MM-DD.md）
  → 日志硬门 → 清除规模声明 → 切回 PLAN
```

## 信号词判定

```
用户输入
  ├─ 信号词（"实现"/"deal"/用户自定义）
  │    → 切 IMPLEMENT，本次任务做完切回 PLAN
  │
  ├─ 模糊（"搞吧"/"都实现"/"直接改"/"你看着来"）
  │    → 弹 AskUserQuestion：仅本次 / 本次会话 / 永久
  │
  └─ 禁止信号（"别动"/"继续讨论"/"只出方案"）
       → 保持 PLAN，不弹框
```

- 信号词用户自定义。默认作用范围仅本次任务。
- 用户可关绿色通道（任何修改前都弹框）。
- 状态用 CLAUDE.md 标记，不靠记忆。

## 阶段锚定规则

```
没有实现信号词时：
  "加入/加上/同步/更新" → 锚定 PLAN → "加到方案里还是动手？" → 弹框
  "可以" + 追加要求       → 方案方向可以 ≠ 授权动手 → 弹框
  "[动词]完告诉我效果"    → 信息请求 → 不实现，出方案

核心：所有动词默认锚定 PLAN。不存在"隐含的实现授权"。
```

## 反推动计数器

```
Agent 催"开始写吧/可以push了/同步吧" → 用户拒绝 → 计数 +1
用户认可（隐式信号验证）→ 计数 -1
计数 = 2 → 锁死 → 直到用户说信号词才解锁
```
