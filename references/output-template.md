# CLAUDE.md + CLAUDE.d/ 生成模板

## O1：CLAUDE.md（≤200行）

只放"被忘了会出事故"的信息。结构：

```markdown
# {项目名} — 项目配置

【当前模式：PLAN】

## 信号词
- {用户自定义实现信号词} → 切 IMPLEMENT
- 模糊信号（搞吧/都实现/直接改/你看着来）→ 必须弹框确认
- {用户自定义禁止信号词} → 保持 PLAN

## 安全边界
- 操作范围：{I2 答案}
- 隐私：{I1 答案}
- API 调用规则：{I1 答案中的 API 约束}

## 操作确认阈值
- 读文件：{自动 or 先问}
- 写文件：{自动 or 先问}
- 删文件：{自动 or 先问}
- 跑命令：{自动 or 先问}
- Git 操作：{自动 or 先问}
- 安装依赖：{自动 or 先问}
- 环境操作：{谁来建、谁来排查}
- 错误处理：{I13 答案}
- 确认有效期：{仅本次/本次会话/永久}

## 项目决策
- 目标：{I4}
- 技术栈：{I5}
- 技术路线：Phase 1 {...} → Phase 2 {...}
- 项目架构：{I6}
- 数据/目录结构：{实际目录树}
- 版本控制：{I9}

## 代码规范
- {I10 答案}
- 测试：{I11 答案}
- {其他已确认的编码约束}

## 三方库
- {库名 + 官方文档链接}
- 排查链：试修→问手册→没手册→"我去找还是你提供"

## 代码默认规范
- 缩进：Tab | 命名：snake_case + PascalCase
- 类型注解：强制（所有函数参数+返回值）
- 格式化：ruff，pre-commit hook
- import 排序：系统级 → 标准库 → 三方库 → 本地库
- 代码最简：不用类不嵌套函数（非必要不建），重复 ≥3 次才抽函数
- 注释：{用户语言}，功能说明+参数说明
- 功能间空一行；import 与功能间空三行
- 测试文件命名：`功能名_test`

## 日志
- **硬门：不写日志不许 IMPLEMENT→PLAN 切换。**
- 触发：每次改动自动写
- 位置：双线——`logs/agent/` + Git commit message
- 文件：`logs/agent/YYYY-MM-DD.md`
- 格式：JSON，`{"timestamp":"...", "summary":"...", "files":["..."], "impact":"..."}`

## 交互
- 回复风格：{I15}

## 方法论摘要
- 用户没说的必须问，不猜不代选
- 先出方案再动手，模糊信号弹框确认
- S/A/B 信息分级，S 级不过不动手
- 声明能力边界：{M6-E 环境确认结果}
- 遵守反推动计数器

## 参考索引
详细规范见 CLAUDE.d/ 目录：
- CLAUDE.d/tech-stack.md
- CLAUDE.d/architecture.md
- CLAUDE.d/coding-style.md
- ...
```

### 关键约束

- 不放编码风格（I10）、测试细节（I11）、日志格式（I14）等 B 级信息
- 不放不能直接导致事故的信息
- 不超过 200 行。超过就把 B 级项移到 CLAUDE.d/
- 【当前模式：PLAN】必须放在最显眼位置（第 3-5 行）

## O2：CLAUDE.d/ 目录

按需放置细节规范。agent 在需要时自行读取：

```
CLAUDE.d/
├── tech-stack.md       # I5 完整答案 + 版本号 + 兼容性约束
├── architecture.md     # I6 完整答案 + 目录结构图
├── coding-style.md     # I10 完整答案 + 格式化配置
├── toolchain.md        # I7 完整答案 + CI/CD 配置详情
├── testing.md          # I11 完整答案 + 测试框架配置
├── repo-boundary.md    # I8 完整答案 + .gitignore 内容
├── error-handling.md   # I13 完整答案 + 重试策略细节
├── logging.md          # I14 完整答案 + 日志格式模板
└── interaction.md      # I15 完整答案 + 信号词完整列表
```

### 文件内容要求

每个文件 ≤150 行。结构：
1. 用户的选择（3-5 行摘要）
2. 具体参数/配置（表格或代码块）
3. 边界约束（什么情况下这个选择不适用）

---

## O3：触发方式

- 新项目自动检测：检测到空白目录/无 CLAUDE.md → 主动问用户是否需要初始化
- 手动调用：用户输入 `/init` 或"帮我分析一下这个项目怎么做"
- 模糊需求检测：用户的一次请求信息严重不足 → 主动加载 skill

### 自动检测的具体条件

```
触发（任一满足）：
  - 当前工作目录无 CLAUDE.md
  - 用户说"/init" / "初始化项目" / "帮我梳理一下"
  - 用户的第一条需求信息严重不足（<10 个词 + 无技术栈 + 无架构）
  - 用户说"从零开始做X" / "帮我分析一下X"
```

---

## 生成时机

```
初始化访谈全部完成 → 生成 CLAUDE.md + CLAUDE.d/
后续每次配置变更 → 更新 CLAUDE.md 对应节
Phase 切换 → 更新【当前模式】
```
