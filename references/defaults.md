# 默认参数表

用户可在初始化访谈中覆盖以下任何默认值。未覆盖时使用以下默认值。

---

## 代码层（D1-D18）

| # | 项 | 默认值 |
|---|-----|--------|
| D1 | 缩进 | Tab |
| D2 | 引号 | Agent 根据语言自动决定 |
| D3 | 分号 | 根据语言决定 |
| D4 | 行尾逗号 | 不加 |
| D5 | 最大行宽 | Agent 决定 |
| D6 | 文件最大行数 | 无限制 |
| D7 | 函数最大行数 | 无限制 |
| D8 | 命名规范 | snake_case（变量/函数）+ PascalCase（类） |
| D9 | 命名要求 | 常见英文单词拼接 |
| D10 | 注释语言 | 根据用户语言决定 |
| D11 | 注释内容 | 功能说明 + 参数说明 |
| D12 | 功能间空行 | 函数间空一行；import 与功能代码间空三行 |
| D13 | import 排序 | 系统级库 → 标准库 → 三方库 → 本地库 |
| D14 | 类型注解 | 强注解：所有函数参数和返回值必须写 |
| D15 | 类使用 | 遵循最简代码规则，非必要不创建 |
| D16 | 函数嵌套 | 遵循最简代码规则，非必要不创建 |
| D17 | 重复代码阈值 | 相同逻辑出现 3 次才抽取函数 |
| D18 | 格式化工具 | 按语言自动选（Python=ruff, JS=prettier, Go=gofmt） |

---

## 日志层（D19-D24）

| # | 项 | 默认值 |
|---|-----|--------|
| D19 | 日志触发 | 每次改动自动写 |
| D20 | 日志位置 | 双线：`logs/agent/` 目录 + Git commit message |
| D21 | 日志文件命名 | 按日期：`YYYY-MM-DD.md` |
| D22 | 日志格式 | JSON：`{"timestamp":"...", "summary":"...", "files":["..."], "impact":"..."}` |
| D23 | 日志语言 | 根据用户语言决定 |
| D24 | 日志硬门 | **写完日志才能 IMPLEMENT→PLAN 切换。** 不写日志 = 状态切不回去 |

---

## 环境与工具层（D25-D30）

| # | 项 | 默认值 |
|---|-----|--------|
| D25 | conda 环境命名 | 项目开始前问用户 |
| D26 | 包管理器 | pip + conda |
| D27 | lint 工具 | 按语言默认 |
| D28 | lint 运行方式 | pre-commit hook |
| D29 | 测试框架 | 按语言默认（Python=pytest, JS=jest） |
| D30 | 测试文件命名 | `功能名_test` |

---

## 版本控制层（D31-D36）

| # | 项 | 默认值 |
|---|-----|--------|
| D31 | commit 频率 | 每完成一个功能 |
| D32 | commit message 格式 | Conventional Commits |
| D33 | commit message 谁写 | AI 起草，用户确认 |
| D34 | 默认分支名 | main |
| D35 | push 策略 | 推送前先确认 |
| D36 | 仓库边界 | 推送：源码 + 配置 + README + 环境.txt（如有）。禁止推送：CLAUDE.md、.claude/、logs/、config.ini 等开发相关内容 |

---

## 交互层（D37-D43）

| # | 项 | 默认值 |
|---|-----|--------|
| D37 | 实现信号词 | 实现 / run |
| D38 | 禁止信号词 | 只出方案 / 继续讨论 / 别动 |
| D39 | 回复语言 | 根据用户语言决定 |
| D40 | 回复详略 | 详细 |
| D41 | emoji | 不用 |
| D42 | 审查触发 | 每次实现后自动触发 + 用户可手动触发 |
| D43 | 重构建议触发 | 按需，可配合审查模块 |
