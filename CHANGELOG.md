# Changelog

## [0.3.0] - 2026-06-04

### Added
- **钩子硬化**：3 个平台级硬门，从"软约束文本"升级为"硬约束钩子"
- `scripts/check-plan-mode.py`（PreToolUse）：PLAN 模式无规模声明时 deny 所有 Write/Edit — M5 状态门硬化
- `scripts/check-boundary.py`（PreToolUse）：拦截写到项目目录外的操作 — 项目边界硬化
- Stop 钩子（prompt-type）：代码改动后无日志 → block stop — 日志硬门硬化
- SKILL.md frontmatter 内嵌 hooks 配置（技能激活时自动生效）

### Security
- 所有钩子 fail-open：脚本崩溃/异常 → 操作放行，不锁死用户
- 基于 hookify 和 hook-development 已验证代码改造，非自己设计新系统

## [0.2.2] - 2026-06-04

### Added
- **阶段锚定规则**（`references/gate-protocol.md`）：讨论阶段中的"加入/更新/同步/可以"锚定在 PLAN，不是实现信号。没有信号词时 Agent 必须弹框确认
- **隐式信号解读表新增3行**（`SKILL.md`）："[动词]完告诉我效果"="可以"+追加要求=讨论阶段动词解释

### Fixed
- **偷懒缝隙 4**：Agent 将代码方案写入外部文件（`.claude/plans/`）绕过对话确认 → 代码方案必须在对话呈现
- **认知差异陷阱**：用户说"可以加入但得泛化"→Agent 可能理解为"动手加入文件"→ 现在锚定在 PLAN 阶段，弹框确认

## [0.2.1] - 2026-06-04

### Added
- **文档修改协议**：任何文档类改动一律走副本流程（_draft.md → 用户对比 → 合并/保留/丢弃）。Agent 不直接写入源文件。杜绝"改一行不用走副本"的偷懒路径。
- **修改类型分流**：代码类走完整门控，文档类走副本流程。不再有盲区。

### Changed
- `maintenance-protocol.md`：新增"修改类型分流"和"文档修改协议"两节
- `SKILL.md` M3 节：新增"输出完整性约束"——文档改副本，代码方案完

### Fixed
- **偷懒缝隙 2**：Agent 以"其余不变"代替完整输出 → 代码方案必须完整呈现
- **偷懒缝隙 3**：Agent 直接写源文件跳过用户确认 → 文档一律走副本

## [0.1.0] - 2026-06-04

### Added
- 初始版本：用户与 Agent 之间的翻译层 skill
- **SKILL.md**：核心理念、使用说明、7核心方法、S/A/B分级访谈、15维度概述
- **7个核心方法** (M0-M7)：
  - M0 结构化反馈引导（四要素：在哪/做了什么/什么现象/期望什么）
  - M1 联想与补全（展开用户没意识到的维度）
  - M2 渐进式路线图（拆 Phase，拒绝一把梭）
  - M3 双向约束（决策原子性约束 + 选项中性化检查）
  - M4 参考优先协议（问用户→搜索→找不到透明告知）
  - M5 混合状态门 + 反推动计数器（信号词=动手，模糊=弹框，禁止=不动）
  - M6 能力边界声明（项目/代码/AI三层边界 + 低数据量语言标记 + 跨语言污染防护）
  - M7 实现后审查与债务管理（/review调用 + 重构安全协议六步骤）
- **15维度分级访谈**：S级4项（隐私/边界/确认阈值/环境路径）、A级9项、B级4项
- **对话哲学**：苏格拉底式探询 + 半隐式方向确认 + 隐式信号解读
- **references/interview-guide.md**：完整访谈指南 + "我不懂"处理协议 + 决策原子性约束
- **references/gate-protocol.md**：状态门完整逻辑 + 反推动计数器机制
- **references/review-protocol.md**：审查触发 + 重构安全协议 + 硬性/软性评判框架
- **references/output-template.md**：CLAUDE.md + CLAUDE.d/ 生成模板
- **references/defaults.md**：43项默认参数（代码/日志/环境工具/版本控制/交互层）
- **硬门机制**：日志写入成为 IMPLEMENT→PLAN 状态切换前置条件
- 环境上下文感知：启动时自动判断运行环境（终端/VSCode/受限）

## [0.2.0] - 2026-06-04

### Added
- **references/maintenance-protocol.md**：后段协议。首版交付后硬门不关，每次改动走同一个最小门控序列
- **规模分级**：微量/小改动/中改动/大改动，每级对应不同的最低问询集
- **发布节点**：触发词检测（上线/发布/开源/发版本）→ 加载 R1-R6 发布子集
- **references/defaults.md**：43项默认参数（代码/日志/环境工具/版本控制/交互）
- I14 从 B 级升 A 级，展开为 6 个子问题（触发/位置/命名/格式/语言/硬门）
- 日志硬门：不写日志不许 IMPLEMENT→PLAN 切换
- CLAUDE.md 规模声明：文件名 + 级别 + 预计改动清单
- output-template 更新：CLAUDE.md 加入代码默认规范 + 日志格式

### Changed
- SKILL.md：新增后段协议引用 + 维护/扩展/发布流程描述
